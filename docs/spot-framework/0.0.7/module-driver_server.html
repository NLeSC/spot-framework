    • driver/server

    Server side filtering

    Implementation of a dataset backed by a server, which in turn uses fi. postgreSQL Fully asynchronous, based on socketIO.

    Most methods below result in a message with the methodName and a data object, containing:

    • datasets and dataview, or dataset
    • filterId or facetId

    Data can be requested by sending getData with dataset and filter ID, on which the server responds with a newData message containing filterId and data.
